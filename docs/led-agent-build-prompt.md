# 构建任务：LED 大屏外贸全链路 AI Agent 系统

## 目标

构建一套以 AI Agent 为核心的外贸业务自动化系统，面向 LED 大屏销售公司。
系统本身就是交付给客户使用的产品，不是评估工具。
LLM 驱动具有自主行动能力的 Agent，不是简单的问答接口。

---

## 业务背景

LED 大屏外贸公司的典型销售链路：

1. 买家通过多个渠道发来询盘（含：屏幕规格、使用场景、数量、目的地）
2. 业务员判断意向 → 报价 → 跟进谈判
3. 买家确认 → 签 PI（形式发票）→ 付款
4. 工厂生产 → 质检 → 出货
5. 海运 / 空运 → 买家清关 → 收货确认

当前全靠人工，核心痛点：多渠道询盘分散漏单、报价不一致、订单进度不透明、跨时区响应慢。

---

## 技术约束

- **后端语言**：Python 3.11，异步优先
- **LLM 接入**：封装统一客户端，支持通过环境变量切换 Anthropic / OpenAI / Gemini
- **Agent 执行必须使用 tool_use 模式**（function calling），不允许直接拼接字符串输出动作结果
- **每次 Agent 执行的完整 tool_use 链路必须持久化**，用于审计和调试
- **渠道**：Email（Gmail/Outlook）、WhatsApp Business API、阿里国际站
- **前端看板**：Next.js，内部使用，展示实时处理状态和人工介入队列
- **容器化**：Docker Compose，一条命令启动完整环境
- 渠道集成支持 stub 模式（无真实 API 凭证时打印到日志），通过环境变量切换，不改代码

---

## 业务规则：自主程度分级

Agent 在每次处理前必须判断自主等级并记录判断依据：

**Level 1 — 全自主，直接执行**
- 新客户 + 标准规格产品 + 询盘金额小
- 订单进度推送、跟进提醒等例行通知

**Level 2 — 生成草稿，等待人工一键发送**
- 老客户或中等金额询盘
- 需要定制规格（不在标准产品目录）
- 客户有过投诉记录
- 涉及付款条件修改

**Level 3 — 直接转人工，Agent 只做信息整理不起草回复**
- 战略大客户
- 大金额询盘（自行根据业务判断阈值）
- 法律 / 合规 / 认证问题
- 客户要求与人沟通
- Agent 自评置信度低于阈值
- 短时间内同一客户反复升级

## 第一阶段交付物

**目标：走通一条完整的询盘处理闭环，证明 Agent 架构可运行。**

演示场景（写成可执行脚本）：

> 模拟收到 WhatsApp 消息："Hi, we need P4 outdoor LED screen, 20sqm,
> for a shopping mall in Dubai, quantity 1 set, please quote.
> Our budget is around $15,000"

预期执行链：
1. **Classifier Agent**：识别意图为询盘，提取结构化需求（规格、面积、目的地、预算），判断自主等级
2. **Quote Agent**：查产品目录，计算报价（含运费估算、数量折扣），生成报价单
3. **Sales Reply Agent**：用英文起草专业回复（含报价、交期、付款条款）
4. **发送**：stub 模式下打印到控制台
5. 全程 tool_use 链路持久化，看板可见
