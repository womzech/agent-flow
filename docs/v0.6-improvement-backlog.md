## 1. 市场背景速览（2026）

| 信号 | 解读 |
|---|---|
| 中小企业 AI SaaS 渗透率仅 22%；200–1000 人付费上限 ¥8,000/月 | **付费能力存在但天花板低**，必须靠模板复用而非纯人天 |
| 客服场景占企业 AI 需求 45% | **当前 `customer-service` 模板权重应大幅加强** |
| 核心痛点：数据安全、开发门槛、运维、内部协作 | **私有化部署 / 数据脱敏 / 本地模型路由是差异点** |
| 政企采购三类：SaaS 账号 / API 嵌入 / 定制实施 | **AgentFlow 当前只覆盖"定制"，可向"API 嵌入"延伸（PAT 已有）** |
| 行业模板（埃森哲）/ 知识库矢量化（IBM）是企业级模板系统关键 | **vector store + 行业 pack 是 v0.6 重投资方向** |
| 国内政府诊断标准（武汉、东莞）有规定章节 | **诊断报告章节可对齐政府数字化转型诊断模板，争取政府/园区补贴预算** |

## 3. 本 Session 实施清单（11 项 ✅）

按价值密度排序：

1. **I1 prompt caching** — 直接降本，每次诊断省 1–2k input tokens
2. **E10 Anthropic SDK retry/timeout** — 生产可靠性，避免单次抖动失败
3. **D1 模板 TODO 占位符** — price-monitor / excel-batch 不再 demo 翻车
4. **F5 Portal token 过期 + 撤回** — 安全口子收紧
5. **C5 诊断 share_token 访问统计 + 过期 + 撤回** — 同上
6. **F4 数据保留 / 清理脚本** — 旧 token / 失败登录 / 弃稿自动归档
7. **E6 SQLite 备份脚本 + 恢复文档** — ops 基线
8. **E7 Dockerfile + docker-compose.yml** — 一键启动
9. **E4 结构化 JSON 日志** — `console.log` → `log.info/error`，容器化前置

11. **H5 `npm run doctor`** — 检查 env / DB / 迁移 / API key / 端口
