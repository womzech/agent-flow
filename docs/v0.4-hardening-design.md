# v0.4 设计文档：Security & Scale Hardening

> 状态：草案 → 实施中
> 适用版本：AgentFlow v0.4.0

## 1. 目标

v0.3 把产品从单租户升级为多用户 + WeCom 集成。但仍有 8 个企业级 gap：登录可被暴力刷、API 路由无 CSRF、无 PAT 程序化访问、无锁定 / 2FA / 会话管理、列表无分页 / 全局搜索。v0.4 把这些一次性闭合。

非目标（留 v0.5）：

- 行级数据隔离（owner_user_id scoping）
- CSV 批量导入
- 国际化
- Sentry / OpenTelemetry 接入
- 多 corp / SaaS 化

## 2. 数据模型变更（Schema v4）

```sql
-- API tokens (PAT)
CREATE TABLE api_tokens (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id       INTEGER NOT NULL REFERENCES users(id),
  token_hash    TEXT NOT NULL,                -- sha256 hex of plaintext token
  token_prefix  TEXT NOT NULL,                -- first 11 chars "agf_xxxxxxx" for display
  name          TEXT NOT NULL,
  last_used_at  TEXT,
  revoked_at    TEXT,
  created_at    TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE UNIQUE INDEX idx_api_tokens_hash ON api_tokens(token_hash);
CREATE INDEX idx_api_tokens_user ON api_tokens(user_id);

-- Failed login tracking for lockout
CREATE TABLE login_attempts (
  id      INTEGER PRIMARY KEY AUTOINCREMENT,
  email   TEXT NOT NULL,
  ip      TEXT NOT NULL,
  ok      INTEGER NOT NULL,                  -- 1 = success, 0 = fail
  at      TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX idx_login_attempts_lookup ON login_attempts(email, ip, at);

-- Sessions: enable server-side revoke
CREATE TABLE sessions (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id      INTEGER NOT NULL REFERENCES users(id),
  jti          TEXT UNIQUE NOT NULL,         -- random; matches cookie payload "sid"
  ip           TEXT NOT NULL DEFAULT '',
  user_agent   TEXT NOT NULL DEFAULT '',
  last_used_at TEXT NOT NULL DEFAULT (datetime('now')),
  revoked_at   TEXT,
  created_at   TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX idx_sessions_user ON sessions(user_id);
CREATE INDEX idx_sessions_jti ON sessions(jti);

-- 2FA TOTP secrets stored on users
ALTER TABLE users ADD COLUMN totp_secret TEXT;
ALTER TABLE users ADD COLUMN totp_enabled INTEGER NOT NULL DEFAULT 0;

-- FTS5 search index across textual fields
CREATE VIRTUAL TABLE search_index USING fts5(
  kind UNINDEXED,       -- "lead" | "client" | "diagnostic" | "project"
  ref_id UNINDEXED,     -- entity id
  title,                -- short label
  body,                 -- searchable content
  tokenize='unicode61'
);
-- triggers below keep search_index in sync (lib/search.ts)
```

`SCHEMA_VERSION = 4`. `schema_migrations` row inserted on first open.

## 3. Rate Limiting

### 设计

Token bucket per `(route, key)` in-memory `Map`:

```ts
interface Bucket { tokens: number; refilledAt: number }
const STORE = new Map<string, Bucket>();
```

API:

```ts
ratelimit(req, { route: "login", key: ipOrUser, limit: 5, windowMs: 900_000 })
  → { ok: true } | { ok: false, retryAfterSec: number, remaining: 0 }
```

应用到：

| 路由 | key | limit | window |
|---|---|---|---|
| POST /login | ip + email lowercased | 5 | 15 min |
| POST /api/wecom/callback | global | 100 | 1 min |
| POST /api/diagnostics/[id]/generate | user_id | 10 | 1 hour |
| POST /api/deliverables/[id]/bundle | user_id | 30 | 1 hour |

响应 429 + 头部 `Retry-After`, `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`。

### 局限

- 单进程内存：多副本时各自计数（实际部署单机即可，未来抽象到 Redis）。
- 测试：注入 `clock()` 函数实现确定性时间。

## 4. CSRF

Next.js Server Actions 自带 Origin/Referer 验证，无需额外 CSRF token。**风险面在 `/api/*` POST 路由**。

策略：double-submit cookie + HMAC 签名（用现有 `AGENTFLOW_SESSION_SECRET`）

```
1. middleware 检测无 `csrf_token` cookie 时 set 一个新的（base64url(random16).hmac）
2. 客户端在 POST 前读取此 cookie，把值放进 `x-csrf-token` header
3. requireCsrf(req) 校验 cookie 签名 + cookie 值 === header 值
```

应用到：

- POST /api/auth/logout (新增 — GET 改为 405)
- POST /api/diagnostics/[id]/generate
- POST /api/deliverables/[id]/bundle
- PUT /api/blueprints/[id]
- POST /api-tokens/{create,revoke}
- POST /security/sessions/revoke
- POST /security/2fa/{enable,disable}

**例外**：

- `/api/wecom/callback`：已用 4-tuple 签名验证，免 CSRF
- `/api/export/*.csv`：GET 只读，免

PAT 请求（Bearer token）也免 CSRF，因为攻击者无法主动让浏览器发 Bearer header。

## 5. API Tokens (Personal Access Tokens)

- 格式：`agf_<32 random base32 chars>` （44 字符总长，类比 GitHub `ghp_`）
- 创建时：服务端生成 → SHA-256 hash 存库 + plaintext 仅在创建响应里返回**一次**
- 认证：`Authorization: Bearer agf_xxx` → middleware sha256(token) 查 api_tokens.token_hash → 设置 `x-agentflow-user-id`
- 权限：继承 user.role 的所有 permissions（v0.4 不做 token scope，留 v0.5）
- 撤销：设置 `revoked_at`，立即失效（每次请求查 DB）
- 审计：每次成功认证更新 `last_used_at`，每个 endpoint 操作走原有 audit

UI：
- `/api-tokens` 列表 + 新建按钮
- 新建后展示 plaintext 一次（标黄警告："仅显示一次"）
- 撤销按钮

## 6. 账号锁定 + Session 管理

### 锁定

`isLockedOut(email, ip)`：查 `login_attempts` 最近 15 分钟内 `ok=0` 数量 ≥ 5 → 锁定。

实现：

```ts
function isLockedOut(email: string, ip: string): { locked: boolean; until?: Date } {
  const recent = db.prepare(`
    SELECT MAX(at) AS at, COUNT(*) AS c FROM login_attempts
    WHERE ok=0 AND (email = ? OR ip = ?) AND at > datetime('now', '-15 minutes')
  `).get(email, ip);
  if (recent.c >= 5) return { locked: true, until: addMinutes(recent.at, 15) };
  return { locked: false };
}
```

锁定时 login 路由 429 + retry-after header。锁定窗口随便用户名都不能登录这个 IP；用户名维度独立计数（防同时多 IP 攻击）。

### Sessions

- Cookie payload 增加 `sid`（random 16B base64url），同时插入 sessions 表一行。
- `verifySession()` 升级为同时查 `sessions` 表：`jti = sid` AND `revoked_at IS NULL`。
- `/security/sessions` 列出当前用户的活跃 sessions，提供 "撤销" 按钮（设 `revoked_at`）。
- 撤销 own session = 立即被踢出（下一次请求 cookie 校验失败 → /login）。

## 7. 2FA TOTP

- 算法：RFC 6238 HMAC-SHA1, 30 秒窗口, 6 位数字
- secret：20 字节 random，存 `users.totp_secret`（base32 编码）
- 启用流程：
  1. `/security/2fa/setup` 生成 secret，显示 base32 + `otpauth://totp/AgentFlow:<email>?secret=...&issuer=AgentFlow`
  2. 用户在 Authenticator 输入 secret
  3. 输入一个 6 位码验证 → `totp_enabled = 1`
- 登录流程：
  - 第一步表单：email + password
  - 验证成功后 if `totp_enabled` → 第二步：6 位 TOTP 码
  - 6 位码错 → 重新展示第二步表单 + audit `auth.fail` reason=totp
- 禁用：要求当前 TOTP 码 → 清空 secret + enabled=0
- 1 步窗口 slack：当前 30s 窗口前后各 1 个 token 也接受（防时钟漂移）
- QR：**不写 QR 库**。展示 `otpauth://` URI 文本 + base32 secret 让用户手动输入或扫第三方 QR 生成器。后续可以加 QR SVG（30 行代码）。

## 8. 分页 + 全局搜索

### 分页

`parsePagination(sp: URLSearchParams)`:
- `page` (1-based, default 1)
- `pageSize` (default 25, max 200)
- 返回 `{ limit, offset, page }`

repo 层：
- `list(opts: { limit?, offset?, ... }): T[]`
- `count(opts: same): number`

UI 组件 `<Pagination />`：prev/next + `当前页 / 总数` + 跳转。

应用到：leads / clients / projects / tickets / audit / api-tokens / sessions / users。

### FTS5 搜索

新建 SQLite 虚表 `search_index`（FTS5 unicode61 tokenizer）：

```sql
CREATE VIRTUAL TABLE search_index USING fts5(
  kind UNINDEXED, ref_id UNINDEXED, title, body,
  tokenize='unicode61'
);
```

Triggers：

```sql
CREATE TRIGGER leads_ai AFTER INSERT ON leads BEGIN
  INSERT INTO search_index(kind, ref_id, title, body)
  VALUES ('lead', new.id, new.company, ...);
END;
-- ai/au/ad for INSERT/UPDATE/DELETE on each entity
```

UI：
- topbar 添加搜索框 `<form action="/search">`
- `/search?q=...` 按 `kind` 分组展示前 50 条结果，每条 highlight snippet。

权限：搜索结果按当前用户的 permissions 过滤（无 `read:leads` → 不显示 lead 命中）。

## 10. 兼容性

- v0.3 → v0.4：数据库自动迁移（CREATE TABLE IF NOT EXISTS / ALTER TABLE ADD COLUMN）。
- v0.3 已发出去的 session cookie：无 `sid` 字段 → 重新登录后获取。期间用户被踢回 /login，可接受。
- WeCom 配置 / 用户表 / 角色表全部保持不变。
- `AGENTFLOW_SESSION_SECRET` 现在还兼任 CSRF HMAC 密钥；建议在生产环境设置专用值。

## 11. 安全注记

- PAT plaintext 仅显示一次，不入 audit payload。
- TOTP secret base32，仅给 secret 不给 binary；解码错误返回 false。
- 锁定窗口 15 分钟仅基于失败次数；运维可手动清除 `login_attempts`。
- sessions.jti 随机 16B base64url，足以抵抗暴力枚举（96 bits）。
- FTS5 搜索查询走参数化，无 SQL 注入风险。
